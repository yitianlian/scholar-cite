## Run 2 — Playwright path (`--browser`)

```bash
scholar-cite cite "Attention Is All You Need" --format all --limit 1 --browser
```

### What the browser path does

1. Launches a **headful** Chromium via Playwright with light stealth patches
   (hide `navigator.webdriver`, set realistic plugin list, etc.).
2. If Scholar shows the "Please show you're not a robot" page, the tool pauses
   up to 5 minutes for the user to click the challenge in the visible window.
3. Once the search result page renders, the tool parses it directly
   (`<div class="gs_r gs_or ...">`), collects `cluster_id`s, then navigates
   the same browser context to each paper's cite popup.
4. For the 4 export URLs on `scholar.googleusercontent.com`, it switches to
   Playwright's **APIRequestContext** — same cookies, but no download prompt
   (plain `page.goto()` aborts on `text/plain` downloads).
5. On shutdown, cookies are cached at
   `~/.cache/scholar-cite/cookies.json`; subsequent runs skip the captcha.

## What this tells us

- **The Playwright path is the default for reliability**, not a fallback. Plain
  HTTP (including through `scholarly`) will almost always be blocked on a
  fresh IP after the first request or two.
- **Cookies carry us across runs** — one captcha solve unlocks roughly a week
  of silent subsequent queries.
- **RefWorks requires a caveat** — Scholar itself doesn't produce a citation
  string for it; only the redirect URL is available without logging into
  RefWorks.
- **3-second batch delays in the original design are optimistic** — a real
  browser at human-ish pace (1-3s between navigations) is what Scholar
  tolerates.
